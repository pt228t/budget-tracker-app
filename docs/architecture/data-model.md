# Data Model — BudgetPulse
# Google Sheets Backend Schema

**Currency:** INR  
**Workbook:** Separate Google Sheet dedicated to budget tracking  
**Source Integration:** Reads `Recurring_Items` from `joint-spend-automation` workbook  

---

## Sheet: `Budget_Categories`

**Purpose:** Master list of budget categories, auto-synced from joint-spend `Recurring_Items`.

| Column | Type | Description |
|--------|------|-------------|
| `category_id` | String (auto) | Unique identifier (e.g., `cat_grocery`) |
| `category_name` | String | Display name (e.g., "Grocery") |
| `monthly_budget` | Number | Budget limit in INR |
| `source` | Enum | `synced` (from Recurring_Items) or `manual` (user-created) |
| `source_item_name` | String | Original item name in Recurring_Items (for synced) |
| `active_status` | Enum | `Active`, `Paused`, `Archived` |
| `created_month` | String (yyyy-MM) | Month this category was created |
| `notes` | String | Optional notes |

**Sync Logic:**
- Daily Apps Script trigger reads `Recurring_Items` from joint-spend sheet
- Creates new categories for new items, updates `monthly_budget` for changed amounts
- Never deletes — sets `active_status = Archived` if item removed from source
- Preserves user-created (`manual`) categories that don't exist in source

## Sheet: `Sub_Categories`

**Purpose:** User-defined tags under budget categories for granular analytics.

| Column | Type | Description |
|--------|------|-------------|
| `sub_category_id` | String (auto) | Unique identifier |
| `category_id` | String | Parent category reference |
| `sub_category_name` | String | Display name (e.g., "Vegetables", "Dairy") |
| `created_by` | String | Email of user who created it |
| `created_date` | Date | Creation date |

**Rules:**
- No budget limit on sub-categories (analytics/tagging only)
- User can create sub-categories on-the-fly during expense logging
- Same sub-category name can exist under different parent categories

---

## Sheet: `Transactions`

**Purpose:** All expense entries. Core of the tracking system.

| Column | Type | Description |
|--------|------|-------------|
| `transaction_id` | String (auto) | Unique identifier |
| `date` | Date | Transaction date |
| `month` | String (yyyy-MM) | Derived from date for grouping |
| `amount` | Number | Amount in INR |
| `category_id` | String | Budget category reference |
| `sub_category` | String | Optional sub-category name |
| `description` | String | What was purchased (e.g., "BigBasket order") |
| `paid_by` | String | Email of person who paid |
| `funding_source` | Enum | `Joint` or `Personal` |
| `logged_by` | String | Email of person who logged the entry |
| `logged_at` | Timestamp | When the entry was created |
| `modified_at` | Timestamp | Last modification timestamp |
| `notes` | String | Optional notes |

**Rules:**
- `month` is auto-derived from `date` (not manually entered)
- `funding_source` defaults to `Joint` for synced categories
- `paid_by` can differ from `logged_by` (Toshi can log an expense Prashant paid)

## Sheet: `Transactions_YYYY` (Archive)

**Purpose:** Yearly archive for older transactions to prevent active sheet bloat.

| Column | Type | Description |
|--------|------|-------------|
| *(Same as Transactions)* | | All columns identical to `Transactions` |

**Rules:**
- Auto-generated by yearly archive script (e.g., `Transactions_2026`)
- Read-only for historical analytics
- Keeps main `Transactions` tab fast and lean

## Sheet: `Budget_History`

**Purpose:** Historical record of budget amounts per category per month. Enables accurate historical comparison.

| Column | Type | Description |
|--------|------|-------------|
| `month` | String (yyyy-MM) | Budget month |
| `category_id` | String | Category reference |
| `budget_amount` | Number | Budget for that category in that month |
| `source` | Enum | `synced` or `manual` |
| `synced_at` | Timestamp | When this value was synced/set |

**Rules:**
- New row created each month per active category
- If budget changes mid-month, latest value is used (overwrite row for current month)
- Historical months are immutable

---

## Sheet: `Vendor_Patterns`

**Purpose:** Learned vendor-to-category mappings for auto-suggestion.

| Column | Type | Description |
|--------|------|-------------|
| `pattern_id` | String (auto) | Unique identifier |
| `vendor_keyword` | String | Keyword/vendor name (e.g., "BigBasket", "Swiggy") |
| `suggested_category_id` | String | Most frequently used category for this vendor |
| `usage_count` | Number | How many times this mapping was used |
| `last_used` | Date | Last time this suggestion was accepted |

**Rules:**
- Updated every time a user logs a transaction with a matching description
- Suggestion is based on highest `usage_count` for a given keyword
- User can always override — the override creates/updates the pattern
- No hard mapping — same vendor can suggest different categories as usage evolves

## Settlement View (Computed, Not Stored)

**Purpose:** Month-end summary of joint pool health + personal payments.

Computed on-the-fly from `Transactions` + `Budget_History`:

```
Per Category:
  budget_amount = Budget_History[month, category_id].budget_amount
  total_spent = SUM(Transactions[month, category_id].amount)
  joint_spent = SUM(Transactions[month, category_id, funding=Joint].amount)
  personal_spent = SUM(Transactions[month, category_id, funding=Personal].amount)
  variance = budget_amount - total_spent
  status = GREEN | AMBER | RED | CRITICAL

Pool Summary:
  total_budget = SUM(all category budgets for month)
  total_spent = SUM(all transactions for month)
  net_position = total_budget - total_spent
  overspend_categories = categories WHERE variance < 0
  underspend_categories = categories WHERE variance > 0
  
Personal Payment Summary (per person):
  total_personal_paid = SUM(Transactions[month, person, funding=Personal].amount)
  → This is the amount to optionally enter as Direct Payment in joint-spend
```
